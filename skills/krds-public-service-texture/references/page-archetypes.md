# Public-Service Page Archetypes

Use these block recipes when a screen needs more real-service texture.

## Home or Service Overview

Default block recipe:
- main service-entry area
- latest notices or schedules
- support or contact box
- short institutional intro
- frequently used links

Good signals:
- service opening hours or response channel
- service eligibility or key tasks
- recent notice dates

## Notice or Board List

Default block recipe:
- page title and short scope text
- filter toolbar
- total count
- board list with badges and dates
- pagination

Good signals:
- "전체 128건"
- category chips or selects
- important or pinned notice treatment
- date and view count

## Detail Page

Default block recipe:
- title and metadata row
- body content
- attachment download list
- owner or contact box
- related links or return path

Good signals:
- attachment size or type
- 담당 부서
- 문의처
- 최종 수정일

## Application Input

Default block recipe:
- service title and short description
- requirement summary
- step indicator
- form
- help, FAQ, or contact path

Good signals:
- who can apply
- what to prepare
- processing timeline
- support phone number

## Confirm, Complete, or Status

Default block recipe:
- result or confirmation title
- application number or receipt ID
- what happens next
- links to list, status, or related task

Good signals:
- 예상 처리 기간
- 확인 경로
- 수정 가능 여부
- 담당 부서 또는 문의처

## Public Dashboard or Integrated Search

Default block recipe:
- title and scope
- source or update stamp
- key cards or chart area
- definitions or caveats
- related notices or downloads

Good signals:
- 데이터 기준 시점
- 공개 가능 범위
- 항목 정의
- 다운로드 경로
