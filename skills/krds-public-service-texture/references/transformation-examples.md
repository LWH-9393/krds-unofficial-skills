# Transformation Examples

Use these examples when the value of the texture layer needs to be shown concretely.

## Example 1: Public Service Home

### Generic mockup

- Large heading and four equal quick links.
- No visible notices, support path, or ownership block.
- Feels like a demo landing card grid.

### KRDS aligned

- Add skip link, header, footer, and trust markers through [$krds-skill](../krds-skill/SKILL.md).
- Keep the service identity linked to home and the footer structured correctly.
- Preserve a restrained, accessible shell.

Suggested base assets:
- [portal-shell.html](../krds-skill/assets/portal-shell.html)
- [krds-components.css](../krds-skill/assets/krds-components.css)

### Textured public-service screen

- Split the first screen into a service-entry column plus a notice/support column.
- Add latest notices with dates and counts.
- Add a visible support or owner box with department and contact route.
- Keep one concise service-entry intro instead of a promotional hero.

Suggested texture assets:
- [home-service-layout.html](../assets/home-service-layout.html)
- [notice-list-panel.html](../assets/notice-list-panel.html)
- [contact-owner-box.html](../assets/contact-owner-box.html)

## Example 2: Application Input Flow

### Generic mockup

- A bare form with labels and one submit button.
- No staged flow, no help route, and no post-submit expectation.

### KRDS aligned

- Add the step indicator.
- Provide current help-panel structure and contact/help paths.
- Add confirm and complete steps where KRDS requires them.

Suggested base assets:
- [application-form.html](../krds-skill/assets/application-form.html)
- [application-confirm.html](../krds-skill/assets/application-confirm.html)
- [application-complete.html](../krds-skill/assets/application-complete.html)

### Textured public-service screen

- Add a preparation summary or required-documents block near the form.
- Keep a realistic contact path and support wording near the task.
- Make the complete step focus on receipt number, next steps, and status path instead of a generic thank-you line.

Suggested texture assets:
- [contact-owner-box.html](../assets/contact-owner-box.html)
- [attachment-list.html](../assets/attachment-list.html)

## Example 3: Notice List and Detail

### Generic mockup

- Clean card list with title-only links.
- Detail page shows title plus body only.

### KRDS aligned

- Add breadcrumb only on eligible interior pages.
- Use a readable metadata row, clear title hierarchy, and accessible attachments.

Suggested base assets:
- [breadcrumb.html](../krds-skill/assets/breadcrumb.html)
- [board-list.html](../krds-skill/assets/board-list.html)
- [board-detail.html](../krds-skill/assets/board-detail.html)

### Textured public-service screen

- Add filters, result counts, and importance badges to the list.
- Add attachment download metadata, owner info, and return path to the detail page.
- Let the list and detail feel like an operating archive, not a showcase gallery.

Suggested texture assets:
