## What The Document Is For

The source document defines a structured self-check process so that public-sector teams can review whether a digital government service follows the KRDS UI/UX guidance consistently.

Use it as:
- a review framework
- a checklist taxonomy
- a verdict discipline
- a report-writing structure

Do not use it as the only freshness source for live component implementation details.
